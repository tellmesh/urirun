# Refactor & security roadmap

Curated, architecture-level backlog. (`TODO.md` is generated by `prefact` from lint
findings and is regenerated wholesale — strategic items live here so they persist.)
Last updated: 2026-06-23.

## Landed (2026-06-23)

- **Isolated-subprocess result contract hardened.** `urirun.exec` redirects the handler's stdout
  to stderr and emits only the result JSON; the subprocess adapter recovers the last balanced JSON
  object on stdout instead of `{stdout: …}`. Kills a whole bug class (a handler or an imported lib
  — e.g. litellm's banner — printing to stdout broke every `local-function-subprocess` caller).
- **Schema generator handles `**kwargs`/`*args`/`self`.** `model_from_function` skips var-keyword,
  var-positional and the bound receiver; a `**kw` handler yields `additionalProperties: true`
  instead of a phantom `required: ["kw"]`.
- **Manifest drift killed at the tool level.** New `urirun connectors sync-manifest <pkg> [--check]`
  projects `routes`/`uriSchemes`/`adapterKinds` from the decorators into the manifest (AST, no
  import). Run across all 22 Python connectors — fixed real route drift and back-filled
  `uriSchemes` on 12 connectors, restoring scheme discovery / self-healing for them.
- **Route-granular discovery.** `connector_discover` exposes per-connector `routes`;
  `ensure_scheme`/`run_ensuring` take a `route` and prefer the connector covering it (a scheme
  split across connectors, e.g. `fs://`, now resolves to the right package).
- **`node/mesh.py` decomposed further** ~2050 → 1721 L: `node/formatting.py` (CLI tables) and
  `node/task_cli.py` (host task/ticket DSL: `task_command` + the `_task_*` family), re-exported
  from `mesh`.
- **Security — RCE classification hole closed.** `shell://…/command/exec` (a
  `subprocess shell=True` handler) was auto-classified `safe`; added `/command/exec` to the
  denylist and centralized the safe decision (see below). `tests/test_routing.py`.
- **Security — single source of truth.** `node/routing.route_is_safe(uri, declared)` +
  `uri_is_denied(uri)` now back both `safe_route()` and `routes_from_registry()` (no divergence);
  per-binding `config`/`meta` `safe: false` is honored through compile.
- **`node/mesh.py` decomposed** 3099 → ~2050 L into `_util`, `_artifacts`, `paths`, `_version`,
  `routing`, `config`, `transport`, `flow` (re-exported from `mesh`).
- **CLI parser split out of `runtime/v2.py`** 2593 → ~1970 L into `runtime/cli.py`;
  `_build_parser` fan-out 116 → 25. `tests/test_cli_parser.py`.
- **Complexity**: `probe_command` (33), `connector_discover` (29), `_handle_run` (26) split —
  no hard `cc_exceeded` left.
- **Tooling**: `project.sh` duplication scan fixed (`redup` now scans the package, not 7 root
  files); two exact duplicates merged (`load_json`/`write_json`, `keyauth`→`paths`).
- **Tests**: +18 (`test_routing`, `test_cli_parser`, `test_node_extracted`). Suite 334 passed.

## Backlog (owner-level / larger)

1. **Deny-by-default capability model.** Replace the URI-substring denylist with classify-by-
   kind/verb (`read` safe; `command`/`exec`/`admin` safe only when explicitly allowed). This is a
   **behavior change** — it reclassifies many `command` routes safe→unsafe and would break
   `heuristic_flow`, device-agents and autonomous mesh flows. Needs: owner decision, a complete
   safe-verb allowlist, and coordinated flow/test updates. Foundation already in
   `node/routing.route_is_safe`. Consider also making `compile_registry` preserve top-level `safe`
   (today an author must put it under `config`/`meta`).
2. **Deploy hygiene (P0 footguns).** (a) A registry replace-deploy must NOT wipe `authorized_keys`
   (it transiently zeroed `keyCount` and locked out deploy). (b) `--merge` should merge with the
   *served* surface and log dropped routes; add an explicit `--replace`. (c) make `cryptography`
   a hard `urirun[host]` extra.
3. **`_cmd_*` + `main()` → `cli.py`** — only worthwhile AFTER a clean public runtime API exists;
   today the handlers reference ~30 v2 internals (many private), so moving them would relocate, not
   reduce, coupling. The parser→`cli` split was the right boundary; the controller stays with runtime.
4. **Finish `_build_parser` polish (optional, low value)**: `_add_host_subparser` still has fan=45
   (could split `dashboard`/`run`/`ask`); `agent`/`errors`/`compat` are still inline (`_build_parser`
   fan=25 — acceptable).
5. **Runtime god-modules** `runtime/_registry.py` (718 L), `runtime/_scan.py` (666 L),
   `__init__.py` (702 L, fan-in 10) — large but **cohesive cores**, not mixed-concern like the old
   `mesh.py`; split only if a genuine seam appears (low priority).
6. **Desktop abstractions (P1)**: unify the `kvm` connector on a session-aware backend
   (portal/PipeWire capture + `ydotool`/`wtype` input on Wayland, X11 fallback); uniform image-artifact
   contract (CDP returns truncated `base64_head`); a `host-facts` introspection route.
