## Lessons from real operator runs (2026-06-24)

Recent end-to-end work with the chat dashboard, Lenovo node, document sync,
scanner service, artifacts/widgets and `urifix://` exposed the same failure
pattern several times: a natural-language request was understandable, but one
runtime precondition was implicit or only partially checked. The fix is to make
every side-effecting URI flow carry the same loop shape:

```text
intent -> concrete URI flow -> route-level preflight -> execute -> verification contract -> urifix nextIntent/retry
```

Priority improvements:

1. **Contract-first side effects.** Every operation that writes files, restarts a
   service, scans a document, deletes artifacts or deploys routes must return a
   `verification` block with expected/actual counts and named checks. The first
   shared helper is `urirun.host.contracts.file_transfer_verification`; document
   sync now uses it for `document-sync.v1`.
2. **Route-level capability checks.** Never treat a whole scheme as sufficient
   when a workflow needs a specific route. `fs://` being present is not enough
   for document sync; the node must expose `fs://host/file/command/write-b64`
   and usually `fs://host/file/query/read-b64`.
3. **URI object registry.** Dashboard discovery should present registries per
   object: `host`, `node:*`, `service:*`. A selected object must have a visible
   route list and health/lifecycle metadata. The UI now starts doing this; the
   backend should eventually expose the same object model as a formal URI.
4. **Service lifecycle as a first-class URI surface.** `urirun-service-chat` and
   `urirun-service-scanner` should have consistent `query/status`,
   `command/start`, `command/restart`, `command/stop` routes, with a clear owner:
   in-process, port-replace, systemd or docker.
5. **`urifix://` as the standard recovery route.** Known failures should map to
   typed diagnoses: missing node URL, missing auth, missing route, service
   stopped, port busy, failed verification, missing LLM model. The dashboard
   should render the returned `nextIntent` instead of returning generic
   `inspect-error`.
6. **Scanner pipeline split.** Keep the scanner runtime in
   `urirun-service-scanner`; keep document processing in connectors
   (`smartcrop://`, `ocr://`, `docid://`); keep final artifact registration in
   the host artifact registry. Avoid duplicate artifact rows for intermediate
   frames.
7. **Avoid package shadowing regressions.** Tests must cover running from the
   monorepo root and from connector roots, because a local `urirun/` directory
   can shadow the installed package if subprocesses use the wrong cwd.

Near-term extraction targets:

- `host/contracts.py` for verification contracts.
- `host/document_sync.py` for archive sync and node transfer preflight. The
  first split is landed: the transfer loop now lives in `host.document_sync`
  behind a compatibility wrapper in `host_dashboard.py`. The fs transfer route
  matching and fallback shim now live in `host.fs_transfer`.
- `host/service_control.py` for restart/start/status logic. First split landed:
  generic restart command selection/scheduling and chat `port-replace` argv
  building now live there. Port-owner detection and safe replacement helpers
  also moved there behind dashboard compatibility wrappers. Scanner restart
  still stays in the dashboard until its in-process server state is isolated.
- `host/discovery.py` for URI object registry normalization. First split
  landed: node alias resolution, known-node file merge, host-config enrichment
  and node route probing moved out of `host_dashboard.py`.
- `host/object_registry.py` for canonical host/service/node object shapes.
  First split landed: host route rows, host object, service contacts and node
  token annotations moved behind dashboard wrappers.
- `host/scanner_bridge.py` for dashboard-to-scanner API glue. First split
  landed: scanner result registration, document-pdf/camera-scan artifact
  selection and chat attachments moved behind dashboard wrappers.

## Landed (2026-06-23)

- **Isolated-subprocess result contract hardened.** `urirun.exec` redirects the handler's stdout
  to stderr and emits only the result JSON; the subprocess adapter recovers the last balanced JSON
  object on stdout instead of `{stdout: â€¦}`. Kills a whole bug class (a handler or an imported lib
  â€” e.g. litellm's banner â€” printing to stdout broke every `local-function-subprocess` caller).
- **Schema generator handles `**kwargs`/`*args`/`self`.** `model_from_function` skips var-keyword,
  var-positional and the bound receiver; a `**kw` handler yields `additionalProperties: true`
  instead of a phantom `required: ["kw"]`.
- **Manifest drift killed at the tool level.** New `urirun connectors sync-manifest <pkg> [--check]`
  projects `routes`/`uriSchemes`/`adapterKinds` from the decorators into the manifest (AST, no
  import). Run across all 22 Python connectors â€” fixed real route drift and back-filled
  `uriSchemes` on 12 connectors, restoring scheme discovery / self-healing for them.
- **Route-granular discovery.** `connector_discover` exposes per-connector `routes`;
  `ensure_scheme`/`run_ensuring` take a `route` and prefer the connector covering it (a scheme
  split across connectors, e.g. `fs://`, now resolves to the right package).
- **`node/mesh.py` decomposed further** ~2050 â†’ 1721 L: `node/formatting.py` (CLI tables) and
  `node/task_cli.py` (host task/ticket DSL: `task_command` + the `_task_*` family), re-exported
  from `mesh`.
- **Security â€” RCE classification hole closed.** `shell://â€¦/command/exec` (a
  `subprocess shell=True` handler) was auto-classified `safe`; added `/command/exec` to the
  denylist and centralized the safe decision (see below). `tests/test_routing.py`.
- **Security â€” single source of truth.** `node/routing.route_is_safe(uri, declared)` +
  `uri_is_denied(uri)` now back both `safe_route()` and `routes_from_registry()` (no divergence);
  per-binding `config`/`meta` `safe: false` is honored through compile.
- **`node/mesh.py` decomposed** 3099 â†’ ~2050 L into `_util`, `_artifacts`, `paths`, `_version`,
  `routing`, `config`, `transport`, `flow` (re-exported from `mesh`).
- **CLI parser split out of `runtime/v2.py`** 2593 â†’ ~1970 L into `runtime/cli.py`;
  `_build_parser` fan-out 116 â†’ 25. `tests/test_cli_parser.py`.
- **Complexity**: `probe_command` (33), `connector_discover` (29), `_handle_run` (26) split â€”
  no hard `cc_exceeded` left.
- **Tooling**: `project.sh` duplication scan fixed (`redup` now scans the package, not 7 root

## Backlog (owner-level / larger)

1. **Deny-by-default capability model.** Replace the URI-substring denylist with classify-by-
   kind/verb (`read` safe; `command`/`exec`/`admin` safe only when explicitly allowed). This is a
   **behavior change** â€” it reclassifies many `command` routes safeâ†’unsafe and would break
   `heuristic_flow`, device-agents and autonomous mesh flows. Needs: owner decision, a complete
   safe-verb allowlist, and coordinated flow/test updates. Foundation already in
   `node/routing.route_is_safe`. Consider also making `compile_registry` preserve top-level `safe`
   (today an author must put it under `config`/`meta`).
2. **Deploy hygiene (P0 footguns).** (a) A registry replace-deploy must NOT wipe `authorized_keys`
   (it transiently zeroed `keyCount` and locked out deploy). (b) `--merge` should merge with the
   *served* surface and log dropped routes; add an explicit `--replace`. (c) make `cryptography`
   a hard `urirun[host]` extra.
3. **`_cmd_*` + `main()` â†’ `cli.py`** â€” only worthwhile AFTER a clean public runtime API exists;
   today the handlers reference ~30 v2 internals (many private), so moving them would relocate, not
   reduce, coupling. The parserâ†’`cli` split was the right boundary; the controller stays with runtime.
4. **Finish `_build_parser` polish (optional, low value)**: `_add_host_subparser` still has fan=45
   (could split `dashboard`/`run`/`ask`); `agent`/`errors`/`compat` are still inline (`_build_parser`
   fan=25 â€” acceptable).
5. **Runtime god-modules** `runtime/_registry.py` (718 L), `runtime/_scan.py` (666 L),
   `__init__.py` (702 L, fan-in 10) â€” large but **cohesive cores**, not mixed-concern like the old
   `mesh.py`; split only if a genuine seam appears (low priority).
6. **Desktop abstractions (P1)**: unify the `kvm` connector on a session-aware backend
   (portal/PipeWire capture + `ydotool`/`wtype` input on Wayland, X11 fallback); uniform image-artifact
   contract (CDP returns truncated `base64_head`); a `host-facts` introspection route.
