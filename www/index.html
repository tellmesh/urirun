urirun://transports

The same URI runs everywhere it needs to.

A URI names what to run, not how to reach it. Swap the transport without touching the call site — the payload is validated against the registry's JSON Schema first, every time, so behavior stays identical across them all.

in-process

Direct dispatch inside your own process. No network, no server.

HTTP service

POST a URI + payload to a worker. v8_service resolves the host and validates before sending.

gRPC

Unary and streaming calls over a channel for low-latency service meshes.

MCP tool

Every route is projected as an MCP tool with JSON Schema — agents call it like any other tool.

queue

Hand a URI to a worker pool and collect the result asynchronously.

serverless

A single handler turns an event into a URI dispatch — drop it into any function runtime.

Cross-transport agreement is a tested guarantee: the dry-run plan and the executed result are the same whether the call goes in-process, over HTTP, or through gRPC.

urirun://schema

Schema-first command packaging.

Decorate a function and urirun reads its signature into JSON Schema (Draft 2020-12) with Pydantic. The runtime validates input, applies defaults, renders the placeholders, then runs.

fn

Write a function

Types and defaults are the source of truth — no separate parameter file to keep in sync.

{ }

Schema is generated

The signature becomes an inputSchema stored alongside the argv or shell template.

✓

Input is validated

Missing required fields are a schema error before anything executes — in dry-run too.

▶

Placeholders render

Validated values fill {input}, {width}, and the command is built deterministically.

transcode.py
from urihandler.v8 import uri_command

@uri_command("media://local/video/transcode")
def transcode(input: str, output: str,
              width: int = 1280,
              height: int = 720):
    return [
        "ffmpeg", "-i", "{input}",
        "-vf", "scale={width}:{height}",
        "{output}",
    ]

# width/height now carry JSON-Schema
# defaults; payload is checked, then run.

urirun://agents

Your registry is already an agent surface.

The same routes project into an MCP tool list and an A2A agent card — JSON Schema and all. Tool names map back to URIs, so an agent calling a tool is dispatching the registry directly.

project the registry for agents
# expose every route as an MCP tool (JSON Schema included)
python -m urihandler.v8_mcp tools registry.json

# publish an A2A agent card listing the same skills
python -m urihandler.v8_mcp card  registry.json
urirun://quickstart

Scan, validate, compile, run.

Install the package, generate a registry from a project, then dispatch a route. Execution is denied by default — runs are dry until a policy allows them.

install
# Node / JS adapter
npm install urihandler

# Python adapter (module CLI: urirun)
python -m urihandler.v8 --help
build a registry & run a route
# 1 · adopt a project's artifacts
urirun scan ./project \
  --out generated/bindings.v8.json \
  --registry-out generated/registry.json

# 2 · check the contract, list the routes
urirun validate generated/bindings.v8.json
urirun list     generated/registry.json

# 3 · run one URI (dry by default)
urirun run media://local/video/transcode \
  --registry generated/registry.json \
  --payload '{"input":"a.mp4","output":"b.mp4"}'
